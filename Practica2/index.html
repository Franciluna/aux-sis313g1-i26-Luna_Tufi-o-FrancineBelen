PRACTICA N° 2 - GIT,GITHUB - HTML
  1. ¿Qué es una rama (branch) en Git y para qué sirve?

    2. Es una línea de tiempo independiente dentro de tu proyecto. Sirve para desarrollar nuevas funciones, corregir errores o experimentar sin modificar el código principal que ya funciona. Imagina que es como hacer una copia de tu trabajo para probar algo sin miedo a arruinar el original.

  2. ¿Cómo se crea y se cambia entre ramas en Git?

    3. Para trabajar con ramas usas estos comandos:

    • Crear una rama: git branch nombre-de-la-rama
    • Cambiar a una rama: git checkout nombre-de-la-rama (o git switch nombre-de-la-rama en versiones modernas).
    • Crear y cambiar al mismo tiempo: git checkout -b nombre-de-la-rama
  3. ¿Cuál es la diferencia entre la rama main y otras ramas?

    4. La rama main (anteriormente llamada master) es la rama principal, donde vive el código que está listo para producción o que es estable. Las otras ramas son temporales y se usan para el "trabajo sucio"; una vez que el código en ellas es probado y está listo, se integran a la main.

  4. ¿Cómo se combinan ramas en Git (merge)?

    5. Para combinar ramas, usas el comando git merge. Primero te aseguras de estar en la rama que quieres que reciba los cambios (normalmente main) y luego ejecutas git merge seguido del nombre de la rama que quieres fusionar. Git intentará automáticamente combinar los cambios, pero si hay conflictos, tendrás que resolverlos manualmente.

  5. ¿Qué problemas pueden ocurrir al trabajar con ramas (conflictos) y cómo se resuelven?

    6. Los conflictos ocurren cuando dos ramas han modificado la misma parte del código de manera diferente. Para resolverlos, Git marcará los archivos en conflicto y tendrás que abrirlos, revisar las diferencias y decidir qué cambios conservar. Después de resolver el conflicto, debes agregar los archivos resueltos con git add y luego hacer un commit para finalizar la fusión.
